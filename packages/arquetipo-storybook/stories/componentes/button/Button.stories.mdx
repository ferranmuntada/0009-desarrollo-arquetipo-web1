# COMPONENTE BOTÓN

Modulo:

- ButtonModule

```js live=true
import { ButtonModule } from "arquetipo-storybook/dist";
```

Componente:

- Para incorporar el component, el prefijo a utilizar para la etiqueta HTML es:

```html live=true
<mln-button></mln-button>
```

Elemento botón con todas sus variantes. Estas son:

- Button
- Button Primary
- Button Secondary
- Button Is Square Primary
- Button Is Square Secondary
- Button Disabled

Todos los tipos de componentes botón aceptarán los siguientes parámetros:

- text: (String) texto que figurará en el botón.
- classButton: (String) clase que definirá el tipo de botón.
- isSquare: (Boolean) Indica si el botón tendrá o nó el borde redondeado.
- isDisabled: (Boolean) Indica si el botón está deshabilitado o nó.
- type: (String) tipo de botón, por defecto 'submit'
- callback (Subject) Observable que al que se puede subscribir.
```typescript live=true
const callbackObserver = new Subject<boolean>();
callbackObserver.subscribe((d) => console.log(d));

const propsButton = {
    text: "Botón",
    classButton: "primary",
    callback: callbackObserver, // Este callback es un observable al cual nos tenemos que suscribir para recibir para recibir la acción del botón (clic)
};
```
- icon: (string) botón con icono a la derecha - https://material.io/resources/icons/?style=baseline.
- outline: (boolean) El icono con fondo vacío.

Estos parámetros se pueden omitir o añadir en función del botón que se quiera generar

## Button

Botón por defecto que acepta las propiedades:

### Ejemplo de uso:

Definición de modelo de datos del componente:

```js live=true
let text = 'Botón'; 
let classButton = 'primary'; 
let type = 'submit';
let icon = 'delete';
```

Declaración HTML:

```html live=true
<mln-button
  [text]="text"
  [classButton]="classButton"
  [type]="type"
></mln-button>
```

Resultado:

export const Button = () => ({
  component: ButtonComponent,
  props: {
    text: "Botón",
    classButton: "primary",
    callback: testObserver,
  },
  moduleMetadata: {

  },
});
